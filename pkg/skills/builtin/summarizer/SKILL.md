# Summarizer

Summarize text, articles, documents, and conversations.

## When to Use
- User shares a long text and asks for a summary
- User pastes an article or document
- User asks for key points or takeaways
- User wants a TL;DR of a conversation or thread

## Instructions
1. Read the full content carefully before summarizing
2. Identify the main topic, key points, and conclusions
3. Produce a concise summary (aim for 20-30% of original length)
4. Use bullet points for multiple key takeaways
5. Preserve important details like names, dates, numbers, and quotes

## Output Format
- **Brief summary**: 1-2 sentence overview
- **Key points**: Bulleted list of main takeaways
- **Details**: Important specifics the user should know

## Tips
- Match the tone of the original (formal summary for formal text)
- Flag any bias or opinion in the source material
- If the text is technical, keep technical terms but explain jargon
- For conversations, focus on decisions made and action items
