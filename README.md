## About AgentX Network

AgentX Network is a **decentralized AI agent economy platform** ‚Äî infrastructure where autonomous AI agents can operate, earn, and coordinate value as independent economic entities.

Built from the ground up in **Go** for maximum efficiency ‚Äî the most lightweight AI agent runtime ever created.

### What's New in v0.3.0 ‚Äî Nexus Engine

- **Real-time streaming** ‚Äî responses appear word-by-word, not as a wall of text
- **Parallel tool execution** ‚Äî file reads, web searches, and skill lookups run concurrently
- **Native function calling** ‚Äî no more fragile regex parsing of LLM output
- **Unified provider layer** ‚Äî Anthropic, OpenAI, Google, and 12+ vendors through one abstraction
- **Provider failover** ‚Äî automatic fallback with cooldown when primary model errors
- **Intelligent context compression** ‚Äî auto-summarize on overflow, preserving credentials and system context
- **Smart Telegram formatting** ‚Äî markdown tables, long messages, and streaming edits all work cleanly
- **7,400+ lines of legacy provider code deleted** ‚Äî cleaner, faster, more maintainable

### Setup in 2 Minutes

**1. Initialize** ‚Äî run the interactive wizard:

```bash
agentx onboard
```

The wizard walks you through picking an AI provider (OpenAI, Anthropic, Ollama, etc.) and an optional chat channel (Telegram, Discord, etc.).

```bash
agentx onboard --provider ollama                  # local model, no key needed

```

**2. Chat**

```bash
agentx agent -m "What is 2+2?"
```

That's it ‚Äî two steps. The wizard handles config, workspace, and the gateway service.

### Security Sandbox

AgentX runs sandboxed by default ‚Äî agents can only access files within the workspace.

| Tool | Restriction |
| --- | --- |
| `read_file` / `write_file` / `edit_file` | Only within workspace |
| `list_dir` / `append_file` | Only within workspace |
| `exec` | Paths must be within workspace |

Dangerous commands are always blocked: `rm -rf`, `format`, `mkfs`, `dd if=`, `shutdown`, fork bombs.

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.agentx/workspace",
      "restrict_to_workspace": true
    }
  }
}
```

### Heartbeat (Periodic Tasks)

Create `HEARTBEAT.md` in your workspace ‚Äî the agent executes tasks every 30 minutes:

```markdown
# Periodic Tasks
- Check my email for important messages
- Search the web for AI news and summarize
```

## Docker

# Clone & first run
git clone https://github.com/Agentx-network/agentx.git && cd agentx

# Configure
vim docker/data/config.json

# Start
docker compose -f docker/docker-compose.yml --profile gateway up -d

## Uninstall

To completely remove AgentX from your system:

```bash
agentx uninstall
```

This removes:
- The systemd gateway service (`~/.config/systemd/user/agentx-gateway.service`)
- All config, auth, and workspace data (`~/.agentx/`)
- The `agentx` binary itself

Use `--yes` / `-y` to skip the confirmation prompt:

```bash
agentx uninstall --yes
```
